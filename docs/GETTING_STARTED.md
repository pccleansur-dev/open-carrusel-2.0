# Open Carrusel: Complete Setup Guide

This guide is for a clean community install on a new machine.

## 1. Requirements

You need:

- Windows, macOS, or Linux
- Node.js 20 or newer
- npm

Optional:

- Claude Code CLI if you want Claude as the in-app agent
- Codex CLI / OpenAI API key if you want Codex inside Docker
- Docker Desktop if you prefer containerized runs
- A Make scenario + Instagram Business setup if you want one-click publishing

## 2. Project structure you should care about

- `src/`: app code
- `data/`: local JSON data
- `public/uploads/`: logos and reference images
- `.env`: optional Docker / publishing / Codex configuration
- `.env.local`: local Claude CLI path generated by setup

## 3. Local install

### Step 1: install dependencies

```bash
npm install
```

### Step 2: seed clean local files

```bash
npm run setup
```

This creates:

- `data/brand.json`
- `data/carousels.json`
- `data/integrations.json`
- `data/templates.json`
- `data/staged-actions.json`
- `data/style-presets.json`
- `data/exports/`
- `public/uploads/`

### Step 3: start the app

```bash
npm run dev
```

Open:

- [http://localhost:3000](http://localhost:3000)

## 4. Docker install

### Step 1: create `.env`

Copy the example file:

```bash
cp .env.example .env
```

On Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

### Step 2: fill only what you need

Possible variables:

- `OPENAI_API_KEY`
- `MAKE_INSTAGRAM_WEBHOOK`
- `IG_USER_ID`
- `AI_PROVIDER`
- `CODEX_MODEL`

If you are not using publishing yet, you can leave the webhook fields empty.

### Step 3: run Docker

```bash
docker compose up -d --build open-carrusel
```

Open:

- [http://127.0.0.1:3002](http://127.0.0.1:3002)

## 5. AI setup

## Claude Code

If you want Claude as the in-app assistant:

1. Install Claude Code CLI.
2. Authenticate it.
3. Run `npm run setup`.
4. The setup script tries to detect `claude` automatically.
5. On Windows, if needed, set `CLAUDE_CLI_PATH` in `.env.local`.

Example Windows path:

```text
C:\Users\<your-user>\AppData\Roaming\npm\claude.cmd
```

## Codex

If you want Codex in Docker:

1. Put `OPENAI_API_KEY` in `.env`
2. Optionally set:
   - `AI_PROVIDER=codex`
   - `CODEX_MODEL=...`

## 6. First app configuration

When the app opens, do this:

### Brand setup

Configure:

- brand name
- colors
- heading font
- body font
- optional logo
- style keywords

This affects generation quality a lot.

### Integrations

If you want Instagram publishing:

Configure:

- Make webhook URL
- Instagram User ID

You can do it from the app UI or from `.env` for Docker fallback.

## 7. Publishing and scheduling

## Publish now

The app can:

- export images
- auto-generate caption if missing
- upload images
- send them to Make

## Scheduled publish

Important:

- scheduling is local
- the app/server must be running
- if the computer is off, nothing will trigger

Current behavior:

- the scheduler checks due posts periodically
- a scheduled post may publish a few minutes after the exact selected minute, depending on the cron interval

## 8. Community-safe sharing checklist

Before sharing your own fork or zip:

1. Remove `.env`
2. Remove `.env.local`
3. Remove `.claude/settings.local.json`
4. Remove `data/` real content
5. Remove `public/uploads/` real files
6. Remove `.next/`
7. Remove `node_modules/`
8. Remove `.git/` if you are sharing as a plain folder/zip

This repo already includes a script to generate a clean copy from a working install.

## 9. Create a clean community copy

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\prepare-community-copy.ps1 -TargetPath "C:\Users\<your-user>\Desktop\OpenCarrusel 2.0"
```

That script:

- copies the project without your personal data
- removes local env files
- removes local Claude settings
- creates clean `data/`
- creates empty `uploads/`
- replaces `README.md` with the community version

## 10. Troubleshooting

## Node version error

Use Node 20+.

## Claude not found

Install Claude Code CLI and rerun:

```bash
npm run setup
```

## Docker starts but publishing does not work

Check:

- `.env`
- Make webhook
- IG user id
- Docker logs

## Scheduled publish did not fire

Check:

- the app/server is still running
- the computer was on
- `scheduledAt` is in the future when created
- logs show the scheduler loop

## Thumbnails/uploads fail

Check that:

- `public/uploads/` exists
- the upload route responds
- Docker has the volume mounted

## 11. Recommended onboarding flow for new users

If you share this with the community, tell them to do this in order:

1. Run local install
2. Complete brand setup
3. Create first carousel
4. Generate 3 to 5 slides
5. Check auto-generated caption
6. Export once
7. If needed, configure Make + Instagram
8. Test one manual publish
9. Test one scheduled publish while the app is running
